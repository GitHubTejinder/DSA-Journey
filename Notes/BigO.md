Time Complexity
- defines how much time/operations does the algorithm takes to accompolish the task.

Big-O
- notation that describes how number of operations required to complete the task changes as the input value changes

Best Case
- case where the algorithm accompolish the result in minimum number of operations such as in one go itself

Average Case
- case where its inbetween some value

Worst Case
- case where the algorithm runs maximum number of operations to achieve the result
